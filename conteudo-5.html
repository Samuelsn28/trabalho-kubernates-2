Gerenciamento de Recursos

Definindo CPU e Memória para Containers

No Kubernetes, os recursos de CPU e memória são definidos nos arquivos de configuração (YAML), garantindo que cada container receba a quantidade necessária para executar suas funções sem desperdício.

resources:
  requests:
    memory: "256Mi"
    cpu: "250m"
  limits:
    memory: "512Mi"
    cpu: "500m"

Boas Práticas para Evitar Problemas de Performance

  • Definir limites e requisições adequadas para cada container.
  • Evitar definir valores muito baixos, prevenindo reinicializações constantes.
  • Monitorar o uso de recursos com ferramentas como Prometheus e Grafana.
  • Revisar periodicamente os valores configurados para otimizar o desempenho.
